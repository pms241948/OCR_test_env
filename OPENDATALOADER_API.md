# OpenDataLoader API Guide

This document describes how to call the OpenDataLoader PDF endpoint exposed by this project.

## Endpoint

- Backend direct: `POST /api/ocr/opendataloader`
- Local Docker example: `http://localhost:3001/api/ocr/opendataloader`
- Frontend proxy example: `http://localhost:8080/api/ocr/opendataloader`

## Purpose

This endpoint uploads a PDF to the backend and runs the local `@opendataloader/pdf` parser inside the backend container or backend process.

Important:

- only PDF files are supported
- runtime execution does not require external internet access
- Java is required only on the backend runtime side
- if you use the provided backend Docker image, Java is already included

## Request Format

Content type:

```http
multipart/form-data
```

Form fields:

- `file`: uploaded PDF file
- `config`: JSON string with OpenDataLoader options

## Supported Config Fields

```json
{
  "outputFormats": ["json", "text", "markdown", "html"],
  "keepLineBreaks": false,
  "useStructTree": true,
  "contentSafetyOff": "",
  "replaceInvalidChars": ""
}
```

Field notes:

- `outputFormats`
  - supported values: `json`, `text`, `markdown`, `html`
  - downloads are returned only for the formats you request
- `keepLineBreaks`
  - keeps parser line breaks where available
- `useStructTree`
  - when the PDF contains structure tags, this improves heading, paragraph, list, and table preservation
  - in this project, the default is `true`
- `contentSafetyOff`
  - optional passthrough option for OpenDataLoader safety filters
- `replaceInvalidChars`
  - optional replacement string for invalid characters

## Curl Example

```bash
curl -X POST http://localhost:3001/api/ocr/opendataloader \
  -F 'file=@./sample.pdf' \
  -F 'config={"outputFormats":["json","text","markdown","html"],"keepLineBreaks":false,"useStructTree":true,"contentSafetyOff":"","replaceInvalidChars":""}'
```

## PowerShell Example

```powershell
curl.exe -X POST http://localhost:3001/api/ocr/opendataloader `
  -F "file=@C:\path\sample.pdf" `
  -F "config={""outputFormats"":[""json"",""text"",""markdown"",""html""],""keepLineBreaks"":false,""useStructTree"":true,""contentSafetyOff"":"""",""replaceInvalidChars"":""""}"
```

## JavaScript Example

```ts
const formData = new FormData();
formData.append("file", fileInput.files[0]);
formData.append(
  "config",
  JSON.stringify({
    outputFormats: ["json", "text", "markdown", "html"],
    keepLineBreaks: false,
    useStructTree: true,
    contentSafetyOff: "",
    replaceInvalidChars: "",
  })
);

const response = await fetch("http://localhost:3001/api/ocr/opendataloader", {
  method: "POST",
  body: formData,
});

const payload = await response.json();
console.log(payload);
```

## Response Field Notes

- `content.structuredText`
  - this project reconstructs a structure-oriented text view from OpenDataLoader JSON
  - headings, paragraphs, lists, and tables are normalized for easier reading
- `content.plainText`
  - normalized text view without HTML tags
- `content.html`
  - raw HTML output generated by OpenDataLoader
- `content.markdown`
  - raw Markdown output generated by OpenDataLoader
- `raw`
  - JSON output from OpenDataLoader
- `downloads`
  - downloadable file payloads for the formats you requested
- `historyId`
  - saved history record ID in this application

## Error Cases

Typical failures:

- missing file upload
- non-PDF upload
- Java runtime missing on a non-Docker backend
- OpenDataLoader parser failure

Example error response:

```json
{
  "success": false,
  "error": {
    "message": "OpenDataLoader PDF only supports PDF uploads."
  }
}
```

## Related Endpoints

- `POST /api/run-all`
  - runs the integrated pipeline
- `POST /api/postprocess`
  - sends selected OCR outputs to the postprocess LLM

The OpenDataLoader result from this endpoint can also be passed into the postprocess stage inside this project.
